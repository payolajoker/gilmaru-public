# 길마루 공익 오픈소스 선언문

길마루는 `위치를 더 정확하고 더 쉽게 설명하는 공용 언어`를 만들기 위한 프로젝트다.
우리는 길마루를 특정 기업의 폐쇄형 서비스가 아니라, 누구나 읽고 고치고 복제하고 배포할 수 있는 공익 인프라로 키운다.

## 1. 우리가 만들고 싶은 것

길마루의 목표는 단순하다.

- 주소만으로 설명하기 어려운 현장 포인트를 더 쉽게 찾게 한다.
- 장애인, 고령자, 보호자, 자원봉사자, 지역 커뮤니티가 같은 위치를 더 정확하게 공유하게 한다.
- 공공기관이 아니어도, 누구나 자신의 지역과 공간을 위해 쓸 수 있게 한다.

길마루는 앱 하나로 끝나는 프로젝트가 아니다.

- 길마루 코드 체계
- 오픈 포인트 데이터 포맷
- 오픈 참조 앱
- 오픈 문서와 운영 가이드

이 네 가지를 함께 만드는 공용 인프라다.

## 2. 우리가 지키려는 원칙

### 공익 우선

길마루의 핵심 기능은 시민에게 열려 있어야 한다.
정확한 위치를 찾고 공유하는 기본 기능은 광고나 과금 장벽 없이 제공한다.

### 카피레프트 지향

길마루를 가져다 쓰는 사람은 자유롭게 쓰고 바꿀 수 있어야 한다.
동시에 그 개선이 다시 공동체로 돌아오도록, 가능한 한 강한 공유 조건을 지향한다.

### 접근성 우선

길마루는 `빠른 사람`을 위한 서비스가 아니라 `헤매기 쉬운 사람`을 덜 헤매게 만드는 도구여야 한다.
큰 글씨, 쉬운 문장, 단계형 안내, 키보드 접근성, 저사양 환경 대응을 제품의 본체로 본다.

### 지역 공동체 우선

길마루의 진짜 주체는 중앙 조직이 아니라 지역의 사용자들이다.
장애인단체, 마을 커뮤니티, 축제 운영팀, 학교, 도서관, 쉼터 운영진도 길마루를 직접 쓸 수 있어야 한다.

### 사생활 보호

우리는 사람의 이동을 추적하는 플랫폼이 되지 않는다.
개인 실시간 위치 저장과 과도한 분석은 지양하고, 필요한 최소 정보만 다룬다.

### 벤더 종속 축소

공익 인프라는 특정 상용 지도 공급자에 과도하게 묶여 있으면 약해진다.
현재 기술 스택의 제약은 인정하되, 장기적으로는 더 개방적인 지도와 데이터 스택으로 옮겨갈 수 있어야 한다.

## 3. 길마루가 아닌 것

길마루는 아래 방향을 목표로 하지 않는다.

- 광고 기반 소비자 앱
- 개인 위치 추적 플랫폼
- 폐쇄형 SaaS만을 위한 위치 체계
- 국가 표준을 선점하는 독점 주소 사업
- 데이터를 잠가 두고 운영자만 이익을 얻는 구조

## 4. 우리가 공개하려는 것

길마루가 공익 오픈소스가 되려면 아래가 공개되어야 한다.

- 코드
- 포인트 데이터 구조
- 내보내기 형식
- 운영 가이드
- 접근성 가이드
- 기여 방법

가능하면 아래도 공개한다.

- 샘플 데이터셋
- 출력물 템플릿
- 테스트 시나리오
- 자체 호스팅 가이드

## 5. 우리가 아직 해결하지 못한 것

공익 오픈소스 전환에는 현실적인 숙제가 있다.

- 현재 지도 기능은 상용 지도 SDK 의존성이 있다.
- 단어 데이터와 외부 어휘 파일의 출처와 재배포 가능 여부를 다시 점검해야 한다.
- 스크린샷, 제안서, 시각 자료는 별도 권리관계 확인이 필요하다.

즉, 선언만으로는 충분하지 않다.
권리관계와 기술 의존성을 분리하고 정리해야 진짜 공용 인프라가 된다.

## 6. 운영 방식

길마루는 가능하면 아래 방식으로 운영한다.

- 저장소와 이슈는 공개
- 핵심 결정은 문서와 공개 토론으로 남김
- 기여자는 CLA보다 DCO 서명 방식을 우선 검토
- 지역별 포인트 데이터는 각 커뮤니티가 관리 가능
- 공식 인스턴스는 데모와 참고 구현 역할에 집중

## 7. 지속가능성

공익 프로젝트라고 해서 돈을 받지 말아야 하는 것은 아니다.
다만 돈의 위치가 중요하다.

길마루의 지속가능성은 아래 형태가 맞다.

- 후원
- 유지보수 계약
- 설치 지원
- 교육
- 데이터 정리 대행
- 공동 연구와 공익 프로젝트 협업

핵심 기능을 닫아 놓고 유료화하는 방식은 지양한다.

## 8. 앞으로의 기준

앞으로 길마루에서 새 기능이나 새 제안을 볼 때는 아래 질문을 먼저 던진다.

1. 이 기능이 더 많은 사람에게 열리는가
2. 이 기능이 지역 공동체가 직접 운영할 수 있게 하는가
3. 이 기능이 접근성을 실제로 개선하는가
4. 이 기능이 특정 공급자 종속을 줄이는가
5. 이 기능이 다시 공동체로 환원되는가

이 다섯 질문에 답하지 못하면, 길마루의 공익 오픈소스 방향과 맞지 않는다.
