# Code of Conduct

Gilmaru is a public-interest open source project. We want it to be usable by
local communities, disability advocates, schools, libraries, event teams, and
first-time contributors. That only works if the project stays respectful,
practical, and safe to join.

## Expected behavior

- Discuss ideas and code in good faith.
- Focus on user impact, accessibility, and verifiable facts.
- Explain data sources and validation steps clearly.
- Welcome small contributions, translations, docs fixes, and review feedback.
- Respect people who report accessibility, safety, or field-data problems.

## Unacceptable behavior

- Harassment, insults, dogpiling, or personal attacks
- Discrimination or exclusion based on identity or disability
- Publishing private information without consent
- Deliberately misleading data, fake field reports, or hidden provenance
- Spam, promotion, or issue flooding that blocks project work

## Maintainer response

Maintainers may edit, hide, lock, or remove content that violates this policy.
They may also ask for changes, pause a thread, or temporarily restrict access
when needed to keep the project usable.

## Reporting

For conduct concerns:

- open a private moderation request if the platform supports it
- otherwise open a minimal public issue without personal details and ask for a
  moderator follow-up

If a report includes accessibility or safety harm, say so explicitly. That will
be triaged faster.

## Scope

This policy applies to repository issues, pull requests, docs discussions,
release notes, and project-managed community spaces.
